# 어디로? 제품 요구사항 문서

## 1. 제품 개요

`어디로?`는 여러 미니게임으로 국내 여행 지역과 자동차 여행 코스를 정해 주는 모바일 우선 웹 서비스다. 핵심 사용자는 주말 여행지를 정하지 못한 즉흥 여행자와 커플이다.

핵심 메시지는 다음과 같다.

> 여행, 어디로 가야 하나요?  
> 오늘은 어떤 게임으로 정해볼까요?

MVP는 지도 다트, 여행 슬롯, 행운의 카드 3개 게임으로 출시한다. 새로운 게임은 공통 게임 레지스트리에 등록하여 기존 지역 추첨과 코스 생성 로직을 변경하지 않고 추가할 수 있어야 한다.

## 2. 목표와 성공 지표

- 첫 추첨을 시작한 세션의 최종 코스 확정 완료율 60% 이상을 목표로 한다.
- 게임별 선택률·완료율·재추첨률을 비교한다.
- `게임도 랜덤으로`, 전국 랜덤, 시·도 직접 선택의 이용률을 측정한다.
- 코스 생성 성공률, 카카오맵 열기율, 결과 공유율을 측정한다.
- 개인정보와 정밀 위치는 수집하지 않는다.

## 3. 핵심 사용자 흐름

```text
게임 선택 → 추첨 범위 선택 → 여행지 게임 → 기간 선택 → 코스 생성 → 결과 확정·공유
```

1. 사용자는 지도 다트, 여행 슬롯, 행운의 카드 중 하나를 선택한다.
2. `게임도 랜덤으로`를 누르면 활성 게임 중 하나가 동일 확률로 선택된다.
3. `전국에서 랜덤` 또는 `시·도 먼저 선택`을 선택한다.
4. 전국 랜덤은 시·도와 시·군·구를 차례로 추첨한다.
5. 시·도 먼저 선택은 전국 지도에서 시·도를 직접 고르고 시·군·구만 추첨한다.
6. 처음 선택한 게임을 모든 추첨 단계에서 유지한다.
7. 지역 확정 후 당일 또는 1박 2일 자동차 여행 코스를 생성한다.

## 4. MVP 게임

### 지도 다트

- 조준점이 후보 행정구역 사이를 빠르게 이동한다.
- 화면 탭·클릭 또는 Space·Enter로 다트를 던진다.
- 다트가 확정 지역에 꽂히고 지도와 지역명을 확대 표시한다.
- 실제 지도 면적과 입력 좌표는 당첨 확률에 영향을 주지 않는다.

### 여행 슬롯

- 지역명과 대표 아이콘이 세로로 빠르게 회전한다.
- 정지 입력 후 감속하면서 확정 지역을 공개한다.
- 표시 순서와 확정 결과를 분리한다.

### 행운의 카드

- 여행지를 숨긴 카드를 섞어 보여준다.
- 사용자가 한 장을 선택하면 결과를 공개한다.
- 카드 위치와 표시 개수는 당첨 확률에 영향을 주지 않는다.

## 5. 게임 확장

후속 후보는 컬러 캐치, 여행 룰렛, 랜덤 핀, 여행 캡슐, 나침반, 주사위 여행, 스크래치 지도다. 새 게임은 ID, 이름, 설명, 아이콘, 활성 여부, 노출 순서, 일반 컴포넌트, 동작 줄이기 컴포넌트를 등록한다. 활성 게임만 선택 화면과 자동 선택 대상에 포함한다.

## 6. 지역 선택 규칙

- 전국 17개 시·도와 선택된 시·도의 시·군·구는 각각 동일 확률로 추첨한다.
- 세종처럼 하위 선택지가 하나뿐이면 세부 추첨을 생략한다.
- 제주도는 제주시와 서귀포시를 세부 후보로 사용한다.
- 추첨 시작 후 0.8초 동안 정지 입력을 잠근다.
- 연속 입력이 발생해도 한 번만 결과를 확정한다.
- 앞 단계를 다시 뽑으면 이후 지역과 코스 결과를 초기화한다.

## 7. 재추첨

- 같은 게임으로 시·군·구 다시 뽑기
- 같은 게임으로 전국부터 다시 뽑기
- 다른 시·도 직접 선택하기
- 다른 게임으로 다시 뽑기
- 지역은 유지하고 코스만 다시 만들기

모든 재추첨은 횟수 제한 없이 허용한다.

## 8. 여행 코스

- 당일: 관광·체험 2곳, 식사 1곳, 카페 또는 추가 명소 1곳
- 1박 2일: 관광·체험 4곳, 식사 2곳, 숙소 1곳
- 이동 수단은 자동차로 고정한다.
- 무작위 후보를 뽑은 후 불필요한 왕복을 줄이도록 방문 순서를 재배치한다.
- 후보가 부족하면 같은 시·도의 인접 시·군·구까지 범위를 확장하고 이를 표시한다.
- 후보가 전혀 없으면 임의 장소를 만들지 않고 재추첨을 안내한다.
- 예약, 결제, 실시간 교통·영업 확인, 대중교통 코스는 MVP에서 제외한다.

## 9. 데이터와 외부 서비스

- 관광지·음식점·숙소는 한국관광공사 TourAPI를 기본 데이터로 사용한다.
- 카페와 부족한 후보는 카카오 장소 검색으로 보완한다.
- 장소 상세와 날짜별 자동차 길찾기는 카카오맵으로 연결한다.
- 모든 결과에 `방문 전 영업정보를 확인해 주세요`를 표시한다.
- 외부 API 키는 서버에서만 사용한다.

## 10. 결과와 공유

- 선택 게임, 지역, 기간, 날짜별 일정과 장소 정보를 표시한다.
- `이 게임으로 다시 뽑기`와 `다른 게임 해보기`를 제공한다.
- 로그인 없이 버전이 포함된 공유 URL을 생성한다.
- 다른 기기에서 링크를 열어 같은 코스를 복원한다.
- 삭제된 장소나 손상된 링크는 동일 지역 코스 재생성으로 복구한다.
- 브라우저 세션 동안 현재 진행 상태를 유지한다.

## 11. 디자인과 접근성

- 플레이풀 트래블 스타일을 사용한다.
- 크림 `#FFF9F0`, 하늘 `#4AB5E8`, 코랄 `#FF6B5E`, 노랑 `#FFD447`, 네이비 `#183153`, 그린 `#39B980`을 기본 색상으로 사용한다.
- 둥근 카드, 굵은 외곽선, 부드러운 그림자, 지도·티켓·가방 스티커 모티프를 사용한다.
- 한 화면의 주요 CTA는 하나만 강하게 표시한다.
- 결과는 색상이나 애니메이션에만 의존하지 않고 텍스트로 제공한다.
- 키보드만으로 전체 흐름을 완료할 수 있어야 한다.
- 동작 줄이기 환경에서는 빠른 이동과 회전을 정적 전환으로 대체한다.
- 모바일 360px 화면에서 가로 스크롤이 없어야 한다.

## 12. 인수 조건

- 세 게임이 공통 추첨 결과를 사용한다.
- 네 번째 게임을 레지스트리에 등록할 때 기존 추첨·코스 코드를 수정하지 않는다.
- 전국 랜덤과 시·도 직접 선택이 정상 작동한다.
- 전국 17개 시·도와 하위 행정구역이 올바르게 연결된다.
- 지도 면적, 슬롯 순서, 카드 위치가 당첨 확률에 영향을 주지 않는다.
- 터치·클릭·Space·Enter와 중복 입력 방지가 작동한다.
- 당일과 1박 2일 코스가 정의된 장소 구성을 충족한다.
- API 오류, 데이터 부족, 손상된 공유 링크를 복구할 수 있다.
- 동작 줄이기와 키보드 환경에서도 전체 여정을 완료할 수 있다.
